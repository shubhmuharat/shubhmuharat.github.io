Model — Simplified Mathematical Form

Below are the simplified equations used to approximate POLKUS behaviour for investigative analysis. These are intentionally reduced forms (not production code).

1. Sentiment influence kernel (continuous-time):

\[ I(t) = \int_{-\infty}^{t} K(t - \tau)\, S(\tau)\, d\tau \]
where \(S(\tau)\) is aggregate social signal (volume × polarity) and \(K(\cdot)\) a decaying influence kernel.

2. Policy impact score (discrete):

\[ P_{c,t} = \sigma\!\Big( w_1 I_t + w_2 E_{c,t} + w_3 D_{c,t} + b\Big) \]
where \(P_{c,t}\) = predicted compliance in constituency \(c\) at time \(t\); \(E\)=economic indicator vector, \(D\)=demographic features; \(\sigma\) = logistic.

3. Control objective (optimization view):

\[ \min_{\theta}\; \sum_{c,t} \ell\big( y_{c,t}, P_{c,t}(\theta) \big) + \lambda \|\theta\|_2^2 \]
Loss \(\ell\) = cross-entropy of desired vs observed behaviours; \(\theta\) parameters of the action policy generator.

Pseudocode — Action Selection Module

# POLKUS: simplified action-selection pseudocode (investigative reimplementation)
procedure SELECT_ACTIONS(state, budget):
  inputs: state = {I_t, E_t, D_c}, budget (resource limit)
  candidates = GENERATE_STRATEGIES(state)
  for each strategy s in candidates:
    score[s] = PREDICT_IMPACT(s, state) - COST(s)
  sorted = SORT_DESC(score)
  chosen = []
  cost = 0
  for s in sorted:
    if cost + COST(s) <= budget:
      chosen.append(s)
      cost += COST(s)
  return chosen

Notes: GENERATE_STRATEGIES() includes information ops, policy nudges, payment targets, and ad-placement schedules (simplified for analysis).
